Create a high-signal digest of production-grade best practices and anti-patterns for modern full-stack software development.

This task is research-only. Do not use local files, repository files, connectors, attachments, synced sources, or any private project context unless explicitly pasted into the prompt. Use only external high-trust sources.

Research scope:
- Architecture and code organization
- Frontend engineering and UI/UX design
- Backend and API design
- Security and privacy
- Testing and quality
- Observability and logging
- Deployment, infrastructure, and operations
- Team workflow and change safety

Source requirements:
- Use only high-trust sources such as official documentation, standards bodies, OWASP, NIST, major cloud/platform architecture guidance, major AI lab or platform docs when relevant, and peer-reviewed or institutional sources.
- Exclude low-trust blogs, SEO articles, anonymous posts, low-authority summaries, and generic content farms.
- Prefer guidance that is stable across stacks and frameworks.
- Include anti-patterns and “do not do this” guidance, not just best practices.
- When different trusted sources disagree, explain the tradeoff briefly.

Writing rules:
- Be concise, specific, and practical.
- Favor actionable guidance over abstract principles.
- Avoid stack-specific advice unless it comes from an especially authoritative source or is widely applicable.
- Prefer checklists and rules over long prose.
- Do not invent internal project context.
- Keep the final digest organized, compact, and reusable as a reference baseline.
- Prioritize guidance that is directly useful for reviewing engineering standards, coding-agent instructions, or repository governance documents.

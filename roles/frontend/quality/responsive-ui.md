# frontend/quality/responsive-ui.md

## Use this file when
The task affects layout, spacing, breakpoints, containers, typography, media, or device-specific behavior.

## Rules
- Design for multiple viewport sizes by default.
- Prevent overflow, clipped content, and unusable controls.
- Handle long text, missing media, and dense content safely.
- Preserve touch target usability where relevant.
- Avoid layouts that depend on a single viewport assumption.

## Check for
- Mobile usability
- Tablet and desktop resilience
- Zoom and text scaling tolerance
- Safe behavior with empty, loading, and error states
